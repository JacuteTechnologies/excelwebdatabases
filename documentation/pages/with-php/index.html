ExcelWebDatabases Documentation

PHP Examples, API endpoint URL with parameters

ExcelWebDatabases uses an API with GET requests which returns JSON. The API endpoint is:

The parameters are:

key is your API key. mode is read/write, defined by r or w. Document is the name of the document you would like to access. isMulti is a boolean, on the basis of whether you are accessing an individual cell or multiple cells in one request. cell is only provided if it is a cell request* and is an Excel cell, ex. A4, B1, or a range ex. A1:B2. data is only provided if it is a write request, and is JSON. column and row are only provided if it is a named request* and are names of the row and column that the cell which you are accessing is in.

*ExcelWebDatabases allows you to name rows and columns in your spreadsheet, or leave them unnamed. If your rows and columns are named, you may make requests using the row name and column name of the cell or the cell's name (ex. A1). If your rows and columns are unnamed, you must make the request using the cell's name.

Read requests

When making a read request, you have the option to read a single cell or to read multiple cells.

If you are reading a single cell, you will receive a JSON response with data set to the cell's value. In the API request, set document to the name of the document you are accessing. Set key to your API key. Set isMulti to false. Set mode to r. If your rows and colums are named, you can access the cell by providing row and column names (set row to the row name and column to the column name). You always have the option to provide the cell's name (set cell to the cell's name). PHP examples:

Write requests

When writing to cells, you have the option to write to a single cell or to a range of cells.

When writing to a single cell or multiple cells, you will receive JSON indicating success when successful. In the API request, set document to the name of the document you are accessing. To write to a single cell, set the parameters in the request as follows. Set key to your API key. Set data to the data you want to write to the cell. Set isMulti to false. Set mode to w. If your rows and colums are named, you can access the cell by providing row and column names (set row to the row name and column to the column name). You always have the option to provide the cell's name (set cell to the cell's name). PHP examples:

Set data to a JSON array of the cells, where the top left in the JSON is the top left cell that you put. And of course, set document to the document name, key to your API key, and mode to r.
